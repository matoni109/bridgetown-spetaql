Channel

Queries

channel

Description

Find a channel by ID.

Response

Returns a Channel

Arguments
Name Description
id - GlobalId! The channel's ID.

Example

Query
query channel($id: GlobalId!) {
  channel(id: $id) {
    channelType
    createdAt
    externalId
    id
    identifier
    sharedSecret
    status
    updatedAt
  }
}
Variables
{"id": GlobalId}
Response
{
  "data": {
    "channel": {
      "channelType": "SHOPIFY",
      "createdAt": ISO8601DateTime,
      "externalId": "4",
      "id": GlobalId,
      "identifier": "xyz789",
      "sharedSecret": "xyz789",
      "status": "Metadata example of `nonRequiredField`",
      "updatedAt": ISO8601DateTime
    }
  }
}

Mutations

accessTokenAuthenticate

Description

Authenticates an access token.

Response

Returns an AccessTokenAuthenticatePayload

Arguments
Name Description
input - AccessTokenAuthenticateInput! Input for authenticating an access token.

Example

Query
mutation accessTokenAuthenticate($input: AccessTokenAuthenticateInput!) {
  accessTokenAuthenticate(input: $input) {
    idToken {
      scopes
      token
      tokenType
    }
    userErrors {
      field
      message
    }
  }
}
Variables
{"input": AccessTokenAuthenticateInput}
Response
{
  "data": {
    "accessTokenAuthenticate": {
      "idToken": IdToken,
      "userErrors": [UserError]
    }
  }
}

Scalar

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Example
4
